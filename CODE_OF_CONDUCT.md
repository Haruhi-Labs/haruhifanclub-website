# 行为准则

> 本准则在 [Contributor Covenant](https://www.contributor-covenant.org/) 的精神基础上精炼而成，
> 适用于 **haruhifanclub** 项目的一切协作空间。

## 我们的承诺

为营造开放、友好、多元、包容的社区，我们作为贡献者与维护者承诺：让参与本项目成为一段不被骚扰的体验，
无论年龄、体型、身体或精神状况、民族、性别认同与表达、经验水平、教育背景、社会经济地位、国籍、外貌、
种族、宗教或性取向如何。

## 我们的准则

**鼓励的行为：**

- 使用友善、包容的语言。
- 尊重不同的观点与经验。
- 平和地接受建设性批评，并优雅地给出建设性反馈。
- 以社区整体利益为先。
- 对其他社区成员保持同理心与耐心，尤其是新人。

**不可接受的行为：**

- 使用性化的语言或图像，以及任何形式的性骚扰或不受欢迎的性关注。
- 挑衅、侮辱或贬损性评论，人身攻击或政治攻击（trolling）。
- 公开或私下的骚扰。
- 未经明确许可，公开他人的私人信息（如住址、邮箱等，俗称 doxxing）。
- 其他在专业场合中合理判断为不当的行为。

## 适用范围

本准则适用于项目的所有空间（仓库 issue / PR / discussions、提交评论、代码评审等），
以及在公开场合代表本项目时的个人行为（如使用项目官方邮箱、官方社交账号，或作为指定代表参加线上线下活动）。

## 维护者的责任

项目维护者有责任明确"可接受行为"的标准，并对任何不可接受的行为采取恰当、公正的纠正措施，
包括但不限于：编辑、删除或拒绝不符合本准则的评论、提交、代码、issue 及其他贡献；必要时临时或永久禁止
任何参与者参与项目。

## 举报

如遇违反本准则的行为，请私下联系项目维护者举报：

> **联系人占位：`@OWNER`（请替换为真实 GitHub 用户名 / 维护者邮箱）。**
> 在仓库正式启用前，此处先用占位符标注，详见 [`docs/COLLABORATION.md`](docs/COLLABORATION.md) 的"替换占位"步骤。

所有举报都会被认真、保密地审阅与处理。维护者有义务为举报人保密。

## 执行

对于被认定违反本准则的行为，维护者将视情节给予相应处理，可能包括：私下提醒、公开警告、临时封禁、
或永久移出社区。维护者承诺对处理结果保持公正一致。

版本 2.1。
